## 1단계 · 필수 프로그램 설치 (15분)

### 1-1. Node.js
https://nodejs.org → **LTS 버전** (왼쪽 초록 버튼) 다운로드 → 설치

설치 확인(터미널에서):
```bash
node -v
```
→ `v20.x.x` 등이 표시되면 정상.

### 1-2. Git
- **Mac**: 터미널에서 `git --version` 입력 → 설치 팝업 뜨면 설치 클릭
- **Windows**: https://git-scm.com 다운로드 → 기본 설정으로 설치

### 1-3. VS Code (추천, 필수 아님)
https://code.visualstudio.com → 다운로드 → 설치

### 1-4. GitHub 계정
https://github.com → **Sign up**

### 1-5. Vercel 계정
https://vercel.com → **Continue with GitHub** 버튼 → GitHub 연동

## 2단계 · 프로젝트 준비 (10분)

### 2-1. 제공된 stockwiki 폴더를 원하는 위치에 복사
(예: 바탕화면 / 문서 폴더)

### 2-2. 터미널 열기
- **Mac**: `Cmd + Space` → "터미널" 입력 → 엔터
- **Windows**: `Win + R` → "powershell" 입력 → 엔터

### 2-3. 프로젝트 폴더로 이동
예를 들어 바탕화면에 `stockwiki` 폴더가 있다면:
```bash
# Mac
cd ~/Desktop/stockwiki

# Windows
cd $env:USERPROFILE\Desktop\stockwiki
```

### 2-4. 패키지 설치 (약 2~3분)
```bash
npm install
```
경고(warning) 메시지는 무시해도 됩니다. 에러(error)가 아니라면 OK.

### 2-5. 로컬에서 테스트
```bash
npm run dev
```
브라우저에서 http://localhost:3000 접속 → 사이트가 뜨면 성공! 🎉

테스트 후 터미널에서 `Ctrl + C` 로 종료.

## 3단계 · GitHub에 코드 올리기 (10분)

### 3-1. GitHub에서 새 저장소 만들기
https://github.com/new 접속
- **Repository name**: `stockwiki`
- **Public** 선택 (Vercel 연동 편리)
- **README, .gitignore, license 체크 모두 해제** ⚠️ 중요
- **Create repository** 클릭

### 3-2. Git 설정 (처음 한 번만)
```bash
git config --global user.name "본인 이름"
git config --global user.email "GitHub 가입 이메일"
```

### 3-3. 프로젝트를 GitHub에 연결
stockwiki 폴더 안에서:
```bash
git init
git add .
git commit -m "first commit"
git branch -M main
git remote add origin https://github.com/본인아이디/stockwiki.git
git push -u origin main
```

> 💡 첫 `git push`에서 권한 오류가 나면 **Personal Access Token** 필요:
> GitHub → Settings → Developer settings → Personal access tokens →
> Tokens (classic) → Generate new token (classic) →
> Note에 "stockwiki", Expiration "No expiration", scope `repo` 전체 체크 →
> Generate → 표시된 토큰을 복사 → git push 시 비밀번호 자리에 붙여넣기

## 5단계 · 도메인 연결 (15~30분, DNS 전파 대기 포함)

### 5-1. Vercel에 도메인 추가
1. Vercel 프로젝트 → **Settings** → **Domains**
2. `stockwiki.kr` 입력 → **Add** 클릭
3. `www.stockwiki.kr` 도 **Add**
4. 화면에 표시되는 DNS 값을 메모

일반적으로:
```
Type: A        Name: @     Value: 76.76.21.21
Type: CNAME    Name: www   Value: cname.vercel-dns.com
```

### 5-2. 도메인 구매처에서 DNS 설정

#### 🅰️ 가비아 (가장 많이 씀)
1. https://my.gabia.com 로그인
2. **My가비아** → **서비스 관리** → **도메인**
3. stockwiki.kr 우측 **관리** 버튼
4. 상단 **DNS 관리** 탭 → **설정** 버튼
5. 레코드 추가:

   | 타입 | 호스트 | 값/위치 | TTL |
   |------|--------|---------|-----|
   | A | @ | 76.76.21.21 | 3600 |
   | CNAME | www | cname.vercel-dns.com. | 3600 |

6. **저장** 클릭

#### 🅱️ 후이즈
- https://domain.whois.co.kr 로그인
- **내 도메인** → stockwiki.kr → **네임서버/DNS 관리** → **DNS 레코드 관리**

#### 🅲 카페24
- https://www.cafe24.com 로그인
- **나의서비스관리** → **도메인** → stockwiki.kr → **DNS 관리**

#### 🅳 Cloudflare Registrar
- Cloudflare 대시보드 → stockwiki.kr → **DNS** → **Records**
- Proxy 상태는 반드시 **회색 (DNS only)** — 주황색이면 Vercel과 충돌

### 5-3. DNS 전파 대기
- 보통 5분~1시간
- Vercel 도메인 페이지에서 **✓ Valid Configuration** 체크 표시가 뜨면 완료
- https://stockwiki.kr 접속 → 사이트 확인! 🚀

## 6단계 · SEO 설정 (5분, 중요!)

### 6-1. Google Search Console 등록
1. https://search.google.com/search-console 접속
2. **속성 추가** → **도메인** 방식
3. `stockwiki.kr` 입력
4. 표시되는 **TXT 레코드**를 도메인 DNS에 추가 (5-2 방법)
5. 확인 완료되면 → 왼쪽 메뉴 **Sitemap**
6. `sitemap.xml` 입력 → **제출**

### 6-2. 초기 색인 요청 (선택)
Search Console에서 핵심 URL 몇 개만 수동 요청하면 노출이 빨라집니다:
- `https://stockwiki.kr/terms/per`
- `https://stockwiki.kr/terms/pbr`
- `https://stockwiki.kr/terms/roe`
- `https://stockwiki.kr/terms/cpi`
- `https://stockwiki.kr/terms/dcf`

## 🆘 문제 해결

### `npm install` 실패
```bash
npm cache clean --force
rm -rf node_modules package-lock.json
npm install
```

### `git push` 권한 오류
위 3-3 섹션의 Personal Access Token 안내 참조

### Vercel 빌드 실패
- Vercel 대시보드 → Deployments → 실패한 배포 → **View Build Logs**
- 대부분 TypeScript 에러 → 에러 메시지 그대로 질문

### 도메인 연결 후 사이트 접속 안 됨
- 최대 24시간까지 DNS 전파 시간 소요 (대부분 10분~1시간)
- Vercel Domains 페이지의 체크 표시 확인
- 터미널에서 `nslookup stockwiki.kr` 로 DNS 조회
